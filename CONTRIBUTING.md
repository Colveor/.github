# Contributing to Colveor

Thank you for your interest in contributing to Colveor! We welcome contributions, ideas, bug reports, and feature requests from the community.

These guidelines apply across the Colveor ecosystem:

| Repository | Purpose |
|------------|---------|
| `colveor` | Project hub and roadmap |
| `colveor-backend` | Backend platform |
| `colveor-flutter` | Flutter framework and packages |
| `colveor-admin` | Administration panel |
| `colveor-cli` | Command-line tools |
| `colveor-docs` | Documentation |
| `colveor-starter` | Starter templates |
| `colveor-examples` | Sample applications |

## Ways to Contribute

- 🐛 Report bugs by opening an issue
- 💡 Suggest features or improvements
- 📝 Improve documentation and examples
- 🔧 Submit fixes and enhancements via pull requests

## Development Workflow

1. **Fork** the relevant repository.
2. **Create a branch** for your change (e.g. `feature/my-feature` or `fix/my-bug`).
3. **Commit** your changes with clear, descriptive messages.
4. **Add tests and documentation** where appropriate.
5. **Open a Pull Request** against the default branch and fill out the PR template.

## Guidelines

In keeping with our principles, please:

- Follow **Clean Architecture** and keep changes **modular by design**.
- Prioritize **developer experience** and clear, readable code.
- Ensure changes are **production ready** and **secure by default**.
- Keep pull requests focused and reasonably small.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## Reporting Security Issues

Please do not open public issues for security vulnerabilities. See our [Security Policy](SECURITY.md) instead.
